## 1. Environment Setup

Run the setup script to install all dependencies:

```bash
bash setup.sh
```

The setup script will:

1. Install **micromamba** package manager
2. Create a conda environment named `slime` with Python 3.12
3. Install CUDA 12.9, PyTorch 2.9.1, and SGLang 0.5.6
4. Install flash-attention, Megatron-Bridge, Megatron-LM
5. Install the slime package
6. Install additional dependencies (polars, datasets, nltk, spacy, etc.)
7. Set up the Gym environment with uv

### Activate the Environment

After setup completes, activate the environment:

```bash
eval "$(micromamba shell hook --shell=bash)"
micromamba activate slime
```

## 2. Data Processing

This section describes how to download and process the training data for SFT and RL.

> **⚠️ Hardware Requirements**
> - **RAM**: 1TB+
> - **Disk**: 1TB+
> - **CPU**: 64+ cores recommended
>
> Data processing is extremely resource-intensive and time-consuming. Ensure you have sufficient resources before proceeding.

### Download Raw Data

First, run the download scripts to fetch all raw datasets:

```bash
cd data
bash download_sft_train.sh
bash download_rl_train.sh
bash download_rl_val.sh
```

This will download all necessary datasets including math, code, science, instruction following, and agent data.

### Process SFT Data

After downloading, process the data for supervised fine-tuning:

```bash
python process_sft_data.py
```

### Process RL Data

Process the data for RL training:

```bash
python process_rl_data.py
```

## 3. Launching Servers

### SandboxFusion Server

SandboxFusion provides a secure sandbox for running and judging code generated by LLMs. It supports multiple languages including Python, C++, Java, Go, Rust, and more.

**Installation:**

```bash
cd ../SandboxFusion

# Install uv if not already installed
curl -LsSf https://astral.sh/uv/install.sh | sh

# Install dependencies
uv sync --python 3.12

# Install Python runtime
cd runtime/python
uv sync --python 3.10

# Download NLTK data
python -c "import nltk; nltk.download('punkt'); nltk.download('stopwords')"

cd ../..

# Create docs directory
mkdir -p docs/build
```

**Launch the server:**

```bash
uv run uvicorn sandbox.server.server:app --host 0.0.0.0 --port 8080 --workers 64
```

The server will be available at `http://localhost:8080`.

### Gym Server (Workplace Assistant)

The Gym server provides RL training environments for LLMs. The workplace assistant environment tests multi-step tool-using capabilities.

**Installation:**

```bash
cd Gym

# Create virtual environment
uv venv --python 3.12
source .venv/bin/activate

# Install dependencies
uv sync
```

**Launch the server:**

```bash
uv run python resources_servers/workplace_assistant/run_resource_server.py
```

The server will start on `http://0.0.0.0:12000`.

## 3. Training Pipeline

### Checkpoint Conversion

Before training, convert the HuggingFace checkpoint to Megatron torch_dist format:

```bash
cd slime
bash scripts/convert_qwen3_4B_base.sh
```

This script:
- Loads the HuggingFace model `Qwen/Qwen3-4B`
- Converts it to Megatron's torch_dist format
- Saves to `$M2RL_ROOT/checkpoints/Qwen3-4B-Base_torch_dist`

### Supervised Fine-Tuning (SFT)

Run SFT training before RL:

```bash
# Run SFT
bash scripts/lanuch_dist.sh scripts/run-qwen3-4B-base-sft.sh
```

The SFT script uses:
- Learning rate: 5e-5 with cosine decay
- Global batch size: 512
- 2 epochs
- Tensor parallel size: 4, Context parallel size: 2

Output checkpoints will be saved to `checkpoints/Qwen3-4B-sft`.

### Single-Task RL Training

After SFT, run RL training for each task domain:

#### Math RL

```bash
bash scripts/launch_dist.sh scripts/train_single_RL/run-qwen3-4B-math.sh
```

#### Code RL

```bash
bash scripts/launch_dist.sh scripts/train_single_RL/run-qwen3-4B-code.sh
```

#### Science RL

```bash
bash scripts/launch_dist.sh scripts/train_single_RL/run-qwen3-4B-science.sh
```

#### Instruction Following RL

```bash
bash scripts/launch_dist.sh scripts/train_single_RL/run-qwen3-4B-if.sh
```

#### Agent RL

```bash
bash scripts/launch_dist.sh scripts/train_single_RL/run-qwen3-4B-agent.sh
```

Each script:
- Loads the SFT checkpoint as reference
- Uses GRPO (Group Relative Policy Optimization)
- Saves checkpoints at specified intervals
- Logs to tensorboard

**Key RL Parameters:**
- Learning rate: 2e-6 (constant)
- Advantage estimator: GRPO
- Samples per prompt: 16
- Rollout batch size: 128

### Multi-Task RL Training

Alternatively, you can train on all tasks simultaneously using multi-task RL:

```bash
bash scripts/launch_dist.sh scripts/run-qwen3-4B-multi-task.sh
```

The multi-task training:
- Blends data from all five task domains (math, code, science, IF, agent)
- Uses `--balance-data` to ensure balanced sampling across tasks
- Trains with a larger global batch size (2048)
- Saves checkpoints to `checkpoints/Qwen3-4B-multi-5-task-32k-grpo-bs128-lr2e_6`

**Key Multi-Task Parameters:**
- Prompt data: `rl_train/multi_task_blend.parquet`
- Rollout max response length: 32768
- Global batch size: 2048
- Uses balanced data sampling across tasks

### Convert to HuggingFace Format

After RL training (either single-task or multi-task), convert the Megatron checkpoints back to HuggingFace format for downstream use (merging, distillation, or inference):

```bash
# Convert single-task RL checkpoints
python tools/convert_torch_dist_to_hf.py \
    --input-dir ../checkpoints/Qwen3-4B-math-rl \
    --output-dir ../checkpoints_hf/Qwen3-4B-math-rl \
    --origin-hf-dir Qwen/Qwen3-4B

python tools/convert_torch_dist_to_hf.py \
    --input-dir ../checkpoints/Qwen3-4B-code-rl \
    --output-dir ../checkpoints_hf/Qwen3-4B-code-rl \
    --origin-hf-dir Qwen/Qwen3-4B

# Similarly for science, IF, and agent checkpoints...

# Or convert multi-task RL checkpoint
python tools/convert_torch_dist_to_hf.py \
    --input-dir ../checkpoints/Qwen3-4B-multi-rl \
    --output-dir ../checkpoints_hf/Qwen3-4B-multi-rl \
    --origin-hf-dir Qwen/Qwen3-4B
```

**Parameters:**
- `--input-dir`: Path to Megatron torch_dist checkpoint
- `--output-dir`: Path to save HuggingFace format checkpoint
- `--origin-hf-dir`: Original HuggingFace model for tokenizer and config files

The converted checkpoints can then be used for:
- Model merging with MergeLM
- Multi-teacher distillation
- Inference with HuggingFace Transformers or vLLM

## 4. Model Merging

After converting checkpoints to HuggingFace format, merge the single-task models using MergeLM:

```bash
cd MergeLM
bash run.sh
```

The merging script supports multiple methods:
- `average_merging` - Simple weight averaging
- `ties_merging` - Task-specific interference elimination
- `sce_merging` - Selective checkpoint merging
- `task_arithmetic` - Task vector arithmetic

**Configuration:**

Edit `run.sh` to set your model paths (use HuggingFace format checkpoints):

```bash
pretrained_model=../checkpoints_hf/Qwen3-4B-sft
math_model=../checkpoints_hf/Qwen3-4B-math-rl
code_model=../checkpoints_hf/Qwen3-4B-code-rl
sci_model=../checkpoints_hf/Qwen3-4B-science-rl
if_model=../checkpoints_hf/Qwen3-4B-if-rl
agent_model=../checkpoints_hf/Qwen3-4B-agent-rl
```

Merged models will be saved to `MergeLM/merged_models/`.

## 5. Multi-Teacher OPD

Multi-Teacher On-Policy Distillation (OPD) distills knowledge from multiple teacher models:

```bash
cd $M2RL_ROOT/slime
bash scripts/multi_teacher_opd.sh
```

**Prerequisites:**
1. All single-task RL checkpoints must be converted to HuggingFace format
2. Teacher model servers must be running on the specified ports:
   - Math teacher: port 1234
   - Code teacher: port 2345
   - Science teacher: port 3456
   - IF teacher: port 4567
   - Agent teacher: port 5678

**Teacher Server Setup:**

Launch each teacher model as an SGLang server (use HuggingFace format checkpoints):

```bash
# Example for math teacher
python -m sglang.launch_server \
    --model-path $M2RL_ROOT/checkpoints_hf/Qwen3-4B-math-rl \
    --port 1234 \
    --mem-fraction-static 0.8
```

The OPD training:
- Uses the SFT model as student initialization
- Queries all teacher servers for log-probabilities
- Distills knowledge through on-policy learning
- Uses advantage estimator: `on_policy_distillation`
